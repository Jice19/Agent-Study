# 总结

## LangGraph通识

最重要的目的不是要大家学会langgraph的语法等等一系列开发手段

而是要去了解langgraph一些基本的术语


把代理以**工作流Workflow**的形式建模为图的方式去进行的智能体开发



Workflow：
流程是固定不变的  
变化的是：通过human in the loop与command进行工作的交接与决策的判断

对比整个工作流来讲 我们关注的核心应该是State状态的管理与监控


如何去构建Workflow与我需要的核心逻辑


图-------状态图
状态State -------------Schema「限制『reducers』」
节点-------功能函数--------接受的参数是什么 我怎么的接收的  批量send
核心节点-----功能函数---------如何干预-------人机交互------- Command
边-----节点与节点之间的流转
持久化------------状态管理
配置-------线程、配置、模型、地址等等 dict[str,Any]
